# Trading Dashboard — Project Constitution

Self-hosted multi-broker, multi-account trading dashboard. Stocks, forex, commodities, indexes, bonds, ETFs, futures, crypto, CFD, options, derivatives. FE+BE on IONOS VPS behind Cloudflare; broker gateways + PG-18 on NUC15PRO over WireGuard; heavy AI box (RTX 4080 + 64 GB) for large Ollama models on demand.

## Stack

- **BE:** Python 3.14 · FastAPI · SQLAlchemy 2.0 async · Alembic · Pydantic v2 · asyncpg
- **FE:** React 19 · Vite 7 · TS 6.0 strict · Tailwind v4 (`@theme`) · shadcn/ui (in-repo) · Zustand · Storybook 10 · klinecharts (Phase 9+)
- **Test:** Vitest 4 + RTL 16 (FE); pytest 9 + pytest-asyncio + httpx (BE); Playwright (Phase 5+)
- **Infra:** Redis 7 · PostgreSQL 18 native on Windows NUC (never containerized) · Docker Compose (docker-ce in WSL on NUC) · Cloudflare Tunnel (TLS at edge) + nginx defense-in-depth · CF Access + Google IdP
- **Broker SDKs:** `ib_async` · `futu-api` · `schwabdev` (read-only) · `alpaca-py`
- **AI:** Ollama — 7-8B on NUC, 14-70B on heavy box (WoL)
- **Pkg:** pnpm (FE) + uv (BE); Node 24 LTS via Corepack
- **Lint:** ruff + mypy --strict (BE); ESLint 9 flat + `eslint-plugin-boundaries` + `jsx-a11y` + Stylelint (FE); pre-commit + commitlint

Versioning: latest stable at scaffold time; pin via lockfiles only.

### FE invariants
- **Fonts:** Noto only, self-hosted `.woff2` in `frontend/public/fonts/` (3 Latin + 5 CJK). `unicode-range` + `lang`-driven variants. `langForMarket(exchange)` from `src/services/lang.ts`.
- **Mobile-first:** bottom tab bar (mobile) / sidebar (desktop). Min 2.75 rem touch targets. Tables collapse to cards below `md`.
- **CSS units:** rem/%/vh/vw only — no `px`, no `em`. Stylelint `unit-disallowed-list` enforces.

### FE runtime (Phase 3+)
- **Routing:** TanStack Router file-based under `frontend/src/routes/`. `routeTree.gen.ts` is gitignored, regenerated by `pnpm tsr generate` (auto via typecheck/test). Don't hand-edit.
- **Scoped state:** features use `useActiveStores()`. Never import `@/stores/scoped/*` directly (no-restricted-imports).
- **Service ticker:** `getServices()` lazy. Tests/Storybook call `setTickingEnabled(false)`.
- **Mode toggle:** paper→live needs confirm dialog; live→paper direct. Cold load = paper. Mode on `<body data-mode>`.
- **Vite dev proxy:** `/api` + `/health` → `http://10.10.0.2:8000` (NUC over WG).

Per-broker invariants live in phase-memory files (read before changing those surfaces): `phase4_sidecar_topology.md` (IBKR mTLS+CRL+watchdog), `phase5a_shipped.md` (NLV cache), `phase5b_shipped.md` (trade execution), `phase5c_shipped.md` (modify+bracket+fills), `phase6_futu_topology.md`, `phase7a_schwab_topology.md` (OAuth+two-tier+BackendCallback), `phase7b1_shipped.md` (quote engine+WS gateway), `phase7c_alpaca_topology.md` (in-cluster docker+30-symbol cap), `phase9_shipped.md` (charting v1: bar aggregator + historical store + 45 indicators + drag-modify SL/TP), `phase10a_shipped.md` (risk gate at station 4 — 7 checks + admin CRUD + FE WARN/BLOCK banners across preview/place_order/modify_order), `phase10b1_shipped.md` (position-sizing v0.13.0: 3 methods + VolatilityService singleton + /trade/sizing page + bars_1d CAGG). Don't copy that detail here.

Cross-cutting load-bearing rules:
- **Boundary stripping:** `AccountResponse` to FE = `id, broker_id, alias, mode, currency_base, display_order` only. Never `gateway_label`/`account_number`. `account_id` UUID is the only FE handle; `AccountService._resolve_account` is the single chokepoint.
- **Race-free soft-delete:** `BrokerDiscoverer` only soft-deletes rows whose `last_seen_via = ANY(:healthy_labels)` THIS tick. All-unhealthy → empty predicate → zero deletions.
- **Maintenance:** `app/services/ibkr_maintenance.py` is source of truth. Backend returns `503 + Retry-After` during reset; watchdog skips weekend reset.
- **NUC ops:** `deploy/nuc/` (PowerShell launchers + watchdog + Pester). `provision-and-publish.ps1` rotates mTLS; `revoke-cert.ps1` revokes; `renew-sidecar-mtls.ps1` rolls one at a time.
- **Risk gate (Phase 10a, shipped v0.12.0):** validation pipeline has 5 stations including the risk gate. `app/services/risk_service.py::RiskService.evaluate(ctx, mode)` is the chokepoint, called inline in `orders_service.preview_order`, `orders_service.place_order`, and `orders_service.modify_order` (D2 file-split skipped — gate works inline). 7 checks: kill switches, max-daily-loss, PDT (Redis in-flight counter), cross-broker concentration, BP buffer, sidecar margin preview (asymmetric preview/place_order fail policy). Audit rows in `risk_decisions` on BLOCK (fail-OPEN per spec §4 — `risk_audit_insert_failures_total` Counter). `side` is lowercased at the audit boundary to satisfy the CHECK constraint. Admin surface: `/api/risk/limits` + `/api/risk/decisions` (read, JWT) and `/api/admin/risk-limits` (POST/PUT/DELETE soft-delete) + `/api/admin/accounts/{id}/kill-switch` (UPSERT) with CSRF nonce; cap-edit invalidation via `app_config:invalidate:risk_limits` and `app_config:invalidate:kill_switch` Redis pubsub. FE: `TradeTicketModal` renders WARN (acknowledge gate) + BLOCK rows from `PreviewResponse.risk_warnings`/`risk_blockers` and from 422 `risk_gate_blocked` (`RiskGateBlockedError`). Pages: `/admin/risk` (limits CRUD), `/admin/risk/decisions` (feed), `/admin/accounts` (kill-switch row). In-flight Redis counters require single-replica today; multi-worker locking deferred to Phase 24. Gate gated on `isinstance(db, AsyncSession)` so existing stub-Session tests stay green.
- **Position sizing (Phase 10b.1, shipped v0.13.0):** 3 sizing methods (fixed-fractional, fixed-risk-per-trade, vol-targeted) producing a suggested qty + pre-validated risk-gate verdict. `app/services/position_sizing_service.py::PositionSizingService` is the per-request orchestrator; `app/services/volatility_service.py::VolatilityService` is a lifespan singleton at `app.state.vol_service` that reads `bars_1d` (TimescaleDB continuous aggregate over `bars_1m`, alembic 0038) and Redis-caches realized-vol + ATR. Vol formula is `stddev(log returns) × sqrt(252)` over 14 bars, NOT ATR (architect C2). Endpoints: `POST /api/risk/position-size` (JWT, rate-limited 20/s burst via in-process deque limiter per `(jwt_subject, account_id)`), `GET /api/risk/sizing-defaults/{id}` (JWT), `PUT /api/admin/sizing-defaults/{id}` (JWT admin + CSRF nonce). Sizing defaults persist in `app_config` namespace `risk_sizing`. The sizer's gate-evaluation has no side-effects (PDT mint + audit live in `orders_service` AFTER the gate, not inside `evaluate()`), so the spec's `dry_run` flag was unnecessary. `SizingRequest` accepts EITHER `instrument_id` (BIGINT, from /trade/sizing page) OR `(conid + broker_id)` (from TradeTicketModal which has no instrument_id at hand); server resolves via `InstrumentResolver.find_by_alias`. FE surfaces: collapsible sizing section in `TradeTicketModal` (between TIF and Preview; "Use this size" overwrites qty; sizing-scoped WARN+BLOCK banners with `aria-label="Risk gate {warnings,blockers} (sizing)"`), standalone page at `/trade/sizing` (3-column comparison, URL-persisted search params). 6 Prometheus metrics: `position_sizing_compute_total{method,account_currency,verdict}`, `position_sizing_latency_seconds{method}`, `volatility_cache_{hits,misses}_total`, `position_sizing_vol_unavailable_total`, `position_sizing_admin_writes_total{field}`. Kelly criterion deferred to Phase 19 (depends on strategy-tagged backtest stats).

## Coding conventions

Full rules in [`docs/CONVENTIONS.md`](docs/CONVENTIONS.md). Load-bearing one-liners:
- **Python:** 3.14; type hints everywhere; async-only; structlog (no `print`); no bare `except:`; `except (A, B) as exc:` (Py3 tuple-catch).
- **TS:** strict; no `any` without disable + reason; function components only.
- **SQL:** schema via Alembic only; `snake_case`; `TIMESTAMPTZ`; money in `NUMERIC(20, 8)`.
- **Git:** conventional commits; body ≤100 chars; squash-merge; never commit `.env`/`*.key`/`secrets/*`.

Common dev/lint/migrate/deploy commands: [`docs/COMMANDS.md`](docs/COMMANDS.md). Prod deploy: `./scripts/deploy.sh` (rsync + remote build + nginx reload).

## Security

- Never log API keys/tokens/passwords — structlog redacts via processor in `app/core/logging.py`.
- Broker creds in `app_secrets` (Fernet) from Phase 2. Never in git.
- Postgres reachable only via WG (VPS) and LAN/WSL (NUC). Never public.
- FE never sees broker creds.
- Trade-execution endpoints require confirmation nonce (CSRF).

### Subagent model routing (updated 2026-05-11, Phase 10a.5)

**Coding is split between Codex and local Qwen.** Anthropic subagents do **not** write production code — they review.

**By task character (primary decision axis):**

| Task character | Route to | Why |
|---|---|---|
| Multi-file refactors (≥3 files, cross-cutting renames, file splits) | **Codex** | Maintains coherent context across many files |
| Lua scripts / Redis atomics / narrow vendor-API specifics | **Codex** | Broader training-set coverage of API edges |
| Long-context analysis (full spec + plan + repo) | **Codex** | 256K context utilized fully |
| Self-contained module writes (new file, single class, well-specified) | **Qwen** (local) | Greenfield from focused prompts — 34 t/s, 0-3 small patches per task |
| TDD test writers (tests matching a known spec) | **Qwen** | Mechanical, structured output |
| Schema-driven SQL / Alembic migrations | **Qwen** | Highly structured form |
| Existing-code integration / multi-site judgment in one file | **Opus direct** | Holistic pattern matching; preserves orchestration context |
